
Way of Project Developement :
-----------------------------

1. Define project architecture & Generate Implementation Plan.
2. Define Techstack for project.
3. Ask user: for developemet approach to be followed for the project
   - Test Driven Development(TDD) or Behaviour Driven Development (BDD).
   - Wait for response.
   - Continue based on selection of approach choosen.
4. Develop Backend code.
5. Then Develope Frontend as per the backend functionalities.
5. Intergrate frontend and backend.
6. Do full satck code review to ensure all functionalities are working as per requirement.



# Do not start coding directly, First define project architecture.
# Do not generate Backend and Frontend code simulteneously.

Acceptance criteria :
--------------------

1. java 17 & Springboot 3.2.x and Angular 18
2. Follow Monolithic architecture , proper Transaction management
3. Structured Logging , proper exception handling & Resource closing.
4. Proper input validation for Backend and user meassge for incorrect input.
5. SOLID principles
6. Implement proper Actuator endpoints
7. API documentation using Swagger.
